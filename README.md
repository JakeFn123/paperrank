# Paper Search & Comprehensive Evaluation Agent

## Documentation

- Chinese User Manual: `docs/用户使用手册.md`
- Retrieval & Scoring Deep Dive (CN): `docs/检索与评分系统详解.md`
- Architecture Design Spec (CN): `docs/架构设计说明.md`
- Agentic Framework Refactor (CN): `docs/智能体框架重构说明.md`

This project implements the workflow defined in `论文搜索与综合评估智能体.docx`:

- Accept a full research question (not just keywords)
- Decompose the question into independent sub-queries
- Retrieve papers from Semantic Scholar and arXiv through an independent MCP tooling layer
- Build a local lightweight vector index for full-text evidence retrieval (RAG)
- Perform multi-dimensional relevance scoring:
  - content relevance
  - method relevance
  - timeliness
  - quality signal
  - complementarity
- Produce evidence-backed synthesis instead of a plain list
- Surface disagreements instead of forcing one-sided conclusions
- Use an explicit agentic framework (agent loop + tools + skills + subagents + task system)

## 1. Quick Start

### 1.1 Create environment

```bash
cd /Users/jakefan/paperrank
python3 -m venv .venv
source .venv/bin/activate
pip install -U pip
pip install -r requirements.txt
```

### 1.2 Configure env

```bash
cp .env.example .env
```

Default runs in mock LLM mode (`LLM_BACKEND=mock`) so you can run end-to-end even without API keys.

For real model calls:

```env
LLM_BACKEND=openai
OPENAI_API_KEY=...
OPENAI_MODEL=gpt-5.2
```

Optional:

```env
SEMANTIC_SCHOLAR_API_KEY=...
```

## 2. Run CLI Demo

```bash
python run_demo.py "Do rerankers improve RAG reliability in enterprise QA?"
```

Optional flags:

```bash
python run_demo.py "your question" --source all --per-query-limit 6 --ingest-top-n 6
```

## 3. Run Streamlit UI

```bash
streamlit run ui/streamlit_app.py
```

UI features:

- chat-style research question input
- sidebar search trace logs
- sidebar score snapshot (multi-dimensional)
- main panel comprehensive answer with citations
- evidence table (paper ref + page + snippet)

## 4. MCP Tooling Layer

The independent MCP server is implemented in:

- `mcp_servers/academic_search_server.py`

Tools:

- `search_papers(query, limit, source)`
- `list_arxiv_categories()`

You can run the MCP server alone:

```bash
python run_mcp_server.py
```

## 6. How Requirements Map to Implementation

Requirement from Word doc -> Implementation:

1. Problem understanding and decomposition
- `app/llm.py` -> `plan_question()`
- prompt template: `app/prompts.py` -> `build_decompose_prompt()`

2. Multi-dimensional relevance evaluation
- `app/llm.py` -> `score_paper()` (content/method/timeliness)
- `app/tooling.py` -> `compute_quality_signal()`
- `app/tooling.py` -> `compute_complementarity()`

3. Comprehensive answer rather than list output
- `app/llm.py` -> `synthesize()`
- prompt template: `app/prompts.py` -> `build_synthesis_prompt()`

4. Traceable evidence and disagreement rendering
- `app/rag.py` -> PDF chunk retrieval with page metadata
- `app/rag.py` -> local vector store (`data/chroma/chunks.json`) + cosine retrieval
- `app/pipeline.py` -> evidence propagation and synthesis payload
- synthesis prompt explicitly requires disagreement section

## 7. Notes

- In mock mode, embeddings and reasoning are deterministic local fallbacks for easy demo.
- In OpenAI mode, scoring and synthesis are generated by real LLM calls.
- arXiv calls respect 3-second rate spacing in MCP server implementation.
