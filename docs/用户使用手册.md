本文档面向最终用户与部署者，目标是让你不看源码也能完成：

1. 安装与启动
2. 正确使用 UI / CLI
3. 理解重构后的 Agent Loop、Tools、Skills、Task System 输出
4. 快速排查常见问题

配套文档：

- 架构设计：`docs/架构设计说明.md`
- 检索评分细节：`docs/检索与评分系统详解.md`

## 1. 系统现在具备什么能力

V2 版本采用标准智能体框架，包含：

1. `Agent Loop`：按阶段执行（拆解 -> 检索 -> 评分 -> 综合）
2. `Tools`：统一工具调用层
3. `Skills`：结构化技能规范
4. `SubAgents`：角色化执行（Planner/Retriever/Scorer/Synthesizer）
5. `Task System`：持久化任务看板（可追踪每步状态）

业务能力不变并增强：

- 输入完整研究问题
- 自动拆解子查询
- 检索论文（arXiv / Semantic Scholar）
- RAG 证据驱动评分
- 输出中文综合结论（论文信息保留英文）
- 展示任务轨迹与评分过程

## 2. 环境要求

1. Python `3.8+`（推荐 3.10/3.11）

## 4. 配置 `.env`

```bash
cp .env.example .env
```

常用配置：

| 变量名 | 是否必填 | 默认值 | 说明 |
|---|---|---|---|
| `LLM_BACKEND` | 否 | `mock` | `mock`=本地模拟；`openai`=真实模型 |
| `OPENAI_API_KEY` | openai 模式必填 | 空 | OpenAI key |
| `OPENAI_MODEL` | 否 | `gpt-5.2` | 文本模型 |
| `EMBEDDING_MODEL` | 否 | `text-embedding-3-small` | 向量模型 |
| `SEMANTIC_SCHOLAR_API_KEY` | 否 | 空 | 学术 API key（可选） |
| `SEARCH_TOP_K` | 否 | `8` | 最终候选数量上限 |
| `SUBQUERY_COUNT` | 否 | `3` | 子查询数量 |

推荐：

1. 快速跑通：`LLM_BACKEND=mock`
2. 真实效果：`LLM_BACKEND=openai` + 配置 key

## 6. 如何写出高质量问题

建议写法：

1. 包含对象 + 方法 + 指标 + 场景  
示例：`在企业知识库问答中，cross-encoder reranker 是否能提升 RAG 的准确率与稳定性？`

2. 必要时使用“人工锁定关键概念”  
示例：`retrieval augmented generation, reranking, robustness`

3. 不要只写非常泛的问题  
不推荐：`RAG 怎么样？`

## 7. 结果怎么看

## 7.1 两套分数

1. `query_match_score`  
- 检索阶段相关性分（解释论文为什么被召回）

2. `total`  
- 最终综合评分（用于最终排序）

## 7.2 引用格式

- `[P1]`：论文级引用
- `[P1 p.3]`：页码级证据引用

## 7.3 任务看板意义

每次运行都会生成任务链，例如：

1. PaperRank run
2. Question decomposition
3. Academic retrieval
4. Evidence scoring
5. Final synthesis

你可据此判断流程在哪一步慢或失败。

## 8. Tools 与 Skills（用户视角）

## 8.1 Tools（系统能力调用接口）

| Tool | 功能 |
|---|---|
| `search_batch` | 批量检索论文并返回日志 |
| `ingest_pdf` | 下载并切块 PDF |
| `retrieve_evidence` | 检索证据片段 |
| `score_single` | 单篇论文评分 |
| `synthesize` | 综合生成最终回答 |

## 8.2 Skills（执行规范）

| Skill | 对应角色 | 作用 |
|---|---|---|
| `query-decomposition` | Planner | 控制问题拆解质量 |
| `academic-retrieval` | Retriever | 控制检索召回质量 |
| `evidence-grading` | Scorer | 控制证据与评分可解释性 |
| `synthesis` | Synthesizer | 控制最终报告结构与引用 |

## 9. 常见报错与处理

## Q1: `externally-managed-environment`

使用虚拟环境安装：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

## Q2: `ModuleNotFoundError: No module named 'app'`

从项目根目录启动：

```bash
cd /Users/jakefan/paperrank
source .venv/bin/activate
streamlit run ui/streamlit_app.py
```

## Q3: Semantic Scholar 429 / Too Many Requests

1. 等待后重试
2. 先用 `source=arxiv`
3. 申请 Semantic Scholar API key

## Q4: 检索结果不相关

1. 改写问题更具体
2. 使用人工锁定关键概念
3. 增大 `per_query_limit`
4. 检查“问题理解与拆解”是否跑偏
