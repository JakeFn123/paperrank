# 论文搜索与综合评估智能体 - 用户使用手册（V2.1）

> 目标：让用户直接跑通系统，并理解“问题拆解 -> 检索 -> 评分 -> 证据审查”的完整链路。

配套文档：
- `docs/架构设计说明.md`
- `docs/检索与评分系统详解.md`

---

## 1. 你将获得什么

1. 输入研究问题，自动拆解子查询。  
2. 两阶段检索（高召回 + 重排），候选论文默认上限 30。  
3. 子查询 0 命中时自动触发简化重试。  
4. 评分过程可解释（`query_match_score`, `rerank_score`, `total`）。  
5. 最终中文综合结论（论文题名等保留英文）。  
6. 自动证据审查（`evidence_audit`）。  
7. 可在 UI 手工编辑意图槽位并重跑。

## 5. UI 使用流程（推荐顺序）

1. 在左侧设置：
- `检索来源`（all / semantic_scholar / arxiv）
- `每个子查询返回上限`
- `候选论文上限`（建议 20-30）
- `人工锁定关键概念`

2. 输入研究问题，点击“搜索并评估”。

3. 重点看 5 个标签页：
- `检索结果`：论文流 + `Match` + `Rerank` + 元数据
- `综合结论`：最终回答 + 证据审查指标
- `问题理解与拆解`：研究意图、子查询、**结构化槽位可编辑重跑**
- `评分过程`：逐篇分数与公式贡献
- `Agent Loop`：任务看板与执行轨迹

---

## 6. 槽位编辑与重跑（新）

在“问题理解与拆解”页：

1. 直接编辑以下槽位：
- `subject`
- `intervention`
- `outcome`
- `context`
- `evaluation`

2. 点击“按槽位重跑”。

这会把槽位作为 `intent_slots_override` 送入 Planner，强制影响后续子查询和检索方向。

---

## 7. 三类核心分数怎么读

1. `query_match_score`  
- 检索解释分：概念命中 + 词项相关 + 锁定概念命中。

2. `rerank_score`  
- 两阶段检索第二阶段的重排分（0-100）。

3. `total`  
- 最终排序分（五维加权）。

建议解读顺序：`query_match_score` -> `rerank_score` -> `total`。

---

## 8. 证据审查怎么用（新）

系统会自动输出 `evidence_audit`：
- `passed`: 是否通过审查
- `confidence`: high / medium / low
- `invalid_refs`: 是否出现无效 `[P#]`
- `missing_citation_lines`: 关键结论中缺引用的条目
- `refs_without_evidence`: 引用了无证据片段的论文

如果未完全通过，最终回答会自动追加“证据对齐审查”说明。

## 9. 常见问题

### Q1. 检索结果还是偏少

1. 将 `候选论文上限` 提到 30。  
2. `source` 选 `all`。  
3. 提高 `per_query_limit`（如 10-12）。  
4. 检查是否锁定概念过多、过窄。

### Q2. 关键概念识别不准

1. 在侧栏填写“人工锁定关键概念”。  
2. 在“问题理解与拆解”中手工改 `intent slots` 再重跑。

### Q3. 429 / Too Many Requests

1. 稍后重试。  
2. 优先 `source=arxiv`。  
3. 配置 `SEMANTIC_SCHOLAR_API_KEY`。

### Q4. `ModuleNotFoundError: No module named 'app'`

确保从项目根目录启动：

```bash
cd /Users/jakefan/paperrank
source .venv/bin/activate
streamlit run ui/streamlit_app.py
```
