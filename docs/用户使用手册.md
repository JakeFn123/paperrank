# 论文搜索与综合评估智能体 - 用户使用手册

本文档面向最终使用者与部署者，目标是让你在不看源码的情况下，完成以下工作：

1. 安装与启动系统
2. 使用 CLI / Web UI 进行论文搜索与综合评估
3. 理解输出结果（评分、证据、争议）
4. 快速排查常见报错

如需查看“检索链路与评分公式”的代码对齐解释，请同时阅读：

- `docs/检索与评分系统详解.md`

---

## 1. 系统能做什么

本系统实现了《论文搜索与综合评估智能体》文档中的主链路能力：

1. 接收完整研究问题（不是仅关键词）
2. 自动拆解为多个子查询
3. 从 arXiv / Semantic Scholar 检索论文
4. 进行多维度评分：
- 内容相关性
- 方法相关性
- 时效性
- 质量信号
- 互补性
5. 结合摘要与全文证据（可获取 PDF 时）输出综合回答
6. 在回答中体现文献分歧而非武断单结论

## 4. 配置 `.env`

```bash
cp .env.example .env
```

`.env` 关键项说明：

| 变量名 | 是否必填 | 默认值 | 说明 |
|---|---|---|---|
| `LLM_BACKEND` | 否 | `mock` | `mock` 表示本地模拟推理；`openai` 表示调用真实模型 |
| `OPENAI_API_KEY` | `openai` 模式必填 | 空 | OpenAI API Key |
| `OPENAI_MODEL` | 否 | `gpt-5.2` | 文本生成模型 |
| `EMBEDDING_MODEL` | 否 | `text-embedding-3-small` | 向量化模型（openai 模式） |
| `SEMANTIC_SCHOLAR_API_KEY` | 否 | 空 | Semantic Scholar Key（可选） |
| `SEARCH_TOP_K` | 否 | `8` | 最终保留候选论文上限 |
| `SUBQUERY_COUNT` | 否 | `3` | 问题拆解子查询数量 |

推荐两种模式：

1. 快速演示模式（零密钥）
- `LLM_BACKEND=mock`

2. 真实模型模式
- `LLM_BACKEND=openai`
- 配置 `OPENAI_API_KEY`

## 5. 启动方式

### 5.1 CLI（命令行）

```bash
cd /Users/jakefan/paperrank
source .venv/bin/activate

python run_demo.py "Do rerankers improve RAG reliability?" --source arxiv --per-query-limit 4 --ingest-top-n 2
```

参数说明：

| 参数 | 默认值 | 说明 |
|---|---|---|
| `question` | `Do rerankers improve RAG reliability?` | 研究问题 |
| `--source` | `all` | 检索源：`all` / `arxiv` / `semantic_scholar` |
| `--per-query-limit` | `6` | 每个子查询返回论文数 |
| `--ingest-top-n` | `6` | 前 N 篇尝试下载 PDF 并做证据抽取 |

CLI 输出结构：

1. `PLAN`: 研究意图 + 子查询 + 隐含假设
2. `SEARCH LOG`: 每个子查询命中数
3. `TOP SCORES`: 多维评分结果
4. `FINAL ANSWER`: 综合问答（含引用与分歧）

### 5.2 Web UI（Streamlit）

```bash
cd /Users/jakefan/paperrank
source .venv/bin/activate

streamlit run ui/streamlit_app.py
```

打开浏览器访问终端显示的地址（通常是 `http://localhost:8501`）。

UI 交互说明：

1. 主区：输入研究问题（聊天式）
2. 侧边栏：检索日志、评分概览、运行参数
3. 展开区：Planner 输出、证据表格（引用片段）

### 5.3 独立检索工具服务（MCP 风格 Tool Server）

如需单独运行检索层：

```bash

source .venv/bin/activate

```

默认监听 `127.0.0.1:8765`，可用接口：

1. `GET /health`
2. `GET /tools/list`

`POST /tools/call` 示例：

## 6. 使用建议（高质量结果）

1. 问题尽量具体
- 不推荐：`RAG 怎么样？`
- 推荐：`在企业知识库问答中，reranker 是否显著提升正确率与稳定性？`

2. 先用 `--source arxiv` 验证链路
- arXiv 接口更稳定，便于先确认流程可跑

3. 再尝试 `--source all`
- 增强覆盖率，但遇到单源异常的概率更高

4. 合理设置 `--ingest-top-n`
- PDF 下载/解析是耗时步骤
- 初次建议 `1~3`

## 7. 输出结果如何理解

### 7.1 评分解释

- `content_relevance`: 是否直接回答你的问题
- `method_relevance`: 方法是否适用于你的场景
- `timeliness`: 时间维度是否仍有参考价值
- `quality_signal`: 引用数、来源、作者等辅助质量信号
- `complementarity`: 与其他候选相比是否提供新角度
- `total`: 综合分（排序依据）

### 7.2 `query_match_score` 是怎么计算的

`query_match_score` 是“检索阶段相关性分”，用于解释“为什么这篇论文被留下来”，**不是**最终综合评分 `total`。

计算公式（代码实现）：

```text
query_match_score = concept_hit_count * 1.0 + relevance_score + lock_match_count * 1.5
```

其中：

- `concept_hit_count`
  - 论文命中的“概念组”数量（如 RAG / Reranking / Enterprise QA / Robustness + 你手动锁定的概念标签）。

- `relevance_score`
  - 词项覆盖分 + 技术短语奖励：
  - `coverage = intersection(tokens, anchors) / max(1, min(10, len(anchors)))`
  - `anchors` 来自：用户问题 + 子查询 + 锁定概念
  - 若命中关键短语（如 `retrieval augmented generation`, `reranking`, `robustness`），每个短语额外 `+0.1`。

- `lock_match_count`
  - 论文命中的“人工锁定关键概念”数量。
  - 该项权重更高（`*1.5`），用于强化用户显式指定概念。

锁定概念过滤（防止结果完全跑偏）：

- 当设置了锁定概念后，系统会先做“自适应门槛过滤”：
  - 锁定 1 个概念：至少命中 1 个
  - 锁定 2-3 个概念：至少命中 1 个
  - 锁定 4 个及以上：至少命中 2 个
- 如果过滤后为空，系统会回退到“部分命中候选”中按分数排序，避免 0 结果。

在 UI 中：

- `查询-论文匹配分（命中概念）` 表显示该分值与命中标签。
- `评分过程明细` 展示每篇论文的匹配信息。

### 7.3 证据与引用

- 形如 `[P1]`：论文级引用
- 形如 `[P1 p.3]`：带页码的证据引用（当 PDF 可解析时）

### 7.4 争议部分

输出中的 `Disagreements In Literature` 用于展示不同结论来源，避免“只给单一答案”的偏置。

## 8. 常见问题排查

### Q1: `externally-managed-environment`

原因：你在系统 Python 下直接 `pip install`，被 PEP 668 拦截。

解决：

```bash
python3 -m venv .venv
source .venv/bin/activate
python -m pip install -r requirements.txt
```

### Q2: `ModuleNotFoundError: No module named 'app'`

原因：从错误目录启动，导致项目根目录不在模块路径。

解决：

```bash
cd /Users/jakefan/paperrank
source .venv/bin/activate
streamlit run ui/streamlit_app.py
```

说明：项目入口已做路径自修复，但仍建议从项目根目录启动。

### Q3: `ImportError ... socksio`

原因：环境中启用了 SOCKS 代理，但 `httpx` 缺少 `socksio`。

解决：

```bash
source .venv/bin/activate
python -m pip install socksio
```

### Q4: 检索命中为 0 或命中不相关

建议：

1. 改写问题为更具体的研究问题
2. 先用 `--source arxiv` 检查链路
3. 增加 `--per-query-limit`
4. 将 `LLM_BACKEND` 切到 `openai` 提升问题拆解质量
