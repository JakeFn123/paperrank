# PaperRank 智能体框架重构说明（V2）

## 1. 重构目标

参考 `learn-claude-code` 的标准智能体范式，将原有线性流程重构为显式的 Agent Framework：

1. `agent loop`：可追踪的主循环编排。
2. `tools`：统一注册、统一调用的工具层。
3. `skills`：按需加载的知识技能层。
4. `subagent`：职责隔离的角色执行层。
5. `task system`：持久化任务看板，支持流程状态外置。

## 2. 与参考框架的映射关系

- `s01_agent_loop.py` -> `app/agentic/loop.py`
- `s04_subagent.py` -> `app/agentic/subagents.py`
- `s05_skill_loading.py` -> `app/agentic/skills.py` + `skills/*/SKILL.md`
- `s07_task_system.py` -> `app/agentic/tasks.py`

## 4. Agent Loop 设计

入口：`PaperRankAgentLoop.run(question, options)`

执行顺序：
1. 初始化 root task。
2. Planner SubAgent 产出 `ResearchPlan`。
3. Retrieval SubAgent 通过 tool 检索论文。
4. Scoring SubAgent 执行证据召回与多维评分。
5. Synthesis SubAgent 生成最终回答。
6. 输出 `AgentOutput`（含 task board 快照与 loop trace）。

`AgentOutput` 新增：
- `task_board_snapshot`
- `loop_trace`

## 5. Tools 定义（`app/agentic/tools.py`）

| Tool 名称 | 输入 | 输出 | 说明 |
|---|---|---|---|
| `search_batch` | `question, sub_queries, source, per_query_limit, locked_concepts` | `(papers, search_log)` | 通过 MCP 检索并返回去重候选 |
| `ingest_pdf` | `paper` | `chunk_count` | 下载 PDF + 切块入向量库 |
| `retrieve_evidence` | `question, paper, top_k` | `EvidenceSpan[]` | 从向量库检索证据片段 |
| `score_single` | `question, paper, evidence_snippets, years, complementarity, quality_signal` | `PaperScore` | 计算单篇论文五维分 |
| `synthesize` | `question, papers_payload` | `markdown` | 输出最终中文综合结论 |

## 6. Skills 定义（`skills/*/SKILL.md`）

采用 YAML-like frontmatter：

```md
---
name: query-decomposition
description: ...
tags: planner,query
---
```

当前技能：

| Skill 名称 | 作用 | 关联 SubAgent |
|---|---|---|
| `query-decomposition` | 研究问题拆解、子查询约束 | Planner |
| `academic-retrieval` | 多源检索、召回质量控制 | Retriever |
| `evidence-grading` | 证据召回 + 多维评分 | Scorer |
| `synthesis` | 引用化结论生成 | Synthesizer |

## 7. SubAgent 定义（`app/agentic/subagents.py`）

| SubAgent | 职责 | 输入 | 输出 |
|---|---|---|---|
| `PlannerSubAgent` | 研究意图/子查询拆解 | question, locked_concepts | `ResearchPlan` |
| `RetrievalSubAgent` | 学术检索与日志汇总 | plan, source, per_query_limit | `papers + search_log` |
| `ScoringSubAgent` | RAG 证据 + 五维评分 | papers, ingest_top_n | `ScoredPaper[]` |
| `SynthesisSubAgent` | 综合写作与引用 | scored_papers, plan | final markdown |

## 8. Task System 定义（`app/agentic/tasks.py`）

存储位置：`.paperrank/tasks/task_{id}.json`

任务字段：
- `id`
- `title`
- `assignee`
- `status` (`pending/in_progress/completed/failed`)
- `depends_on`
- `payload`
- `result_summary`
- `error`
- `created_at / updated_at`

典型任务链：
1. `PaperRank run` (orchestrator)
2. `Question decomposition` (planner)
3. `Academic retrieval` (retriever)
4. `Evidence scoring` (evaluator)
5. `Final synthesis` (synthesizer)

## 9. 与现有系统的兼容

- `app/pipeline.py` 保留 `RunOptions` 与 `PaperEvaluationAgent.run` 接口。
- UI 无需改调用方式，已新增可视化区展示：
  - task board
  - loop trace
  - tools / skills 定义

## 10. 后续建议

1. 将 TaskBoard 扩展为支持“重试失败任务/任务回滚”。
2. 为 SubAgent 增加并行执行器（检索与证据提取并发）。
3. 增加 tool 级别指标（耗时、成功率、错误类型）。
4. 引入策略器子代理，按问题类型动态裁剪工具与技能。
