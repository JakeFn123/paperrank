# Synthesis Skill

## 技能说明

该技能负责把多篇论文的评分与证据整合为“可读、可追溯、可审查”的最终结论。  
核心目标不是写长文，而是输出**证据约束的结论**。

## 适用场景

- 需要生成结构化中文综合评估报告。  
- 需要明确文献一致性与分歧。  
- 需要输出引用并通过证据审查。  
- 需要给用户清晰的不确定性说明与下一步确认问题。

## 输入与输出契约

### 输入

- `question`
- `plan`（含 `clarification_questions`）
- `scored_papers`（含 evidence）

### 输出

- `final_answer_markdown`
- `evidence_audit`

## 核心执行步骤

### 阶段 1：参考编号构建

1. 按排序给论文分配 `P1/P2/...`。  
2. 将证据片段绑定到对应 `ref_id`。  
3. 组装传入综合模型的 `papers_payload`。

### 阶段 2：结构化综合写作

输出必须包含以下章节：
1. `## 直接回答`  
2. `## 证据支撑的关键结论`  
3. `## 文献分歧与不同立场`  
4. `## 论文入选原因（多维评分）`  
5. `## 当前不确定性`

### 阶段 3：澄清问题合并

- 若 `plan.clarification_questions` 非空，在报告前置“需你确认的问题”。

### 阶段 4：证据审查（Evidence Audit）

至少检查：
1. 关键结论条目是否带 `[P#]` 引用。  
2. 引用编号是否存在于候选论文编号集合。  
3. 被引用论文是否有可用 evidence 文本。

输出审查结构：
- `passed`
- `confidence`
- `cited_refs`
- `invalid_refs`
- `missing_citation_lines`
- `refs_without_evidence`

### 阶段 5：未通过时追加审查说明

- 当 `passed=false` 时，自动在报告末尾追加 `## 证据对齐审查`，明确问题项。

## 质量门禁（必须通过）

1. 核心结论必须可追溯到至少一条引用。  
2. 不允许出现不存在的引用编号。  
3. 必须显式输出不确定性，不做过度确定性结论。  
4. 默认报告语言为中文，论文元数据保持原文。

## 失败回退策略

1. `scored_papers` 为空时，输出“未检索到可用论文”的最小报告。  
2. 综合模型调用失败时，使用保守模板生成并附不确定性说明。  
3. 审查失败时不隐藏问题，必须显示审查结果。

## 最佳实践

1. 先回答用户问题，再展开证据与分歧。  
2. 引用尽量使用页码级格式（如 `[P1 p.3]`）。  
3. 对高风险结论，至少提供 2 个独立来源支撑。
